# Flutter Boilerplate - Template Usage Guide

## Quick Start
1. Open your AI assistant (Cascade, ChatGPT, etc.)
2. Use the customized prompt from: `comprehensive-report-generator-prompt.md`
3. The AI will analyze your Flutter project and generate reports in the `reports/` directory

## Customized Settings
- **Project**: Flutter Boilerplate
- **Technology**: Flutter/Dart with MVVM Clean Architecture
- **Compliance**: Play Store, App Store
- **Timeline**: Multi-developer team estimations

## Expected Outputs
- `reports/comprehensive-project-report.md` - Markdown version
- `reports/comprehensive-project-report.html` - Professional HTML version

## Template Files Available

### 1. comprehensive-report-generator-prompt.md
Generates comprehensive project analysis reports with:
- Architecture assessment
- Code quality metrics
- Flutter-specific best practices evaluation
- Deployment readiness analysis
- Performance optimization recommendations

### 2. UNIVERSAL_PROJECT_SETUP_PROMPT.md
Creates complete `.llm-context` directory structure for new Flutter projects with:
- Flutter/Dart coding standards
- MVVM architecture guidelines
- Widget testing strategies
- Firebase integration patterns
- Cross-platform deployment guides

## Usage Instructions

### For Project Reports
1. Review the generated template in `.llm-context/templates/comprehensive-report-generator-prompt.md`
2. Customize project-specific details (app name, version, features)
3. Run the analysis with your AI assistant
4. Review and customize the generated reports for stakeholders

### For New Project Setup
1. Use `UNIVERSAL_PROJECT_SETUP_PROMPT.md` for new Flutter projects
2. Adapt the technology stack section for specific dependencies
3. Customize architecture patterns based on project complexity
4. Update compliance requirements for target app stores

## Maintenance
- Update templates quarterly to reflect Flutter SDK updates
- Review generated reports for accuracy with current project state
- Customize compliance sections based on regional requirements
- Adapt architecture recommendations for team size and complexity
